    文档导入 UX 优化计划¶

    📋 目标¶

    优化文档导入功能的用户体验，让用户能够： 1. 选择将文档导入到不同的 Graph（知识库） 2. 创建新的 Graph 来组织不同项目/主题的文档 3. 清楚地看到当前选择的 Graph 和数据去向 4. 管理已有的 Graph（查看、删除）

    🎯 核心问题¶

    当前问题： - ❌ 所有导入都进入 default graph，无法分组管理 - ❌ 用户不知道数据会存储在哪里 - ❌ 多个项目的文档混在一起，无法隔离 - ❌ 无法按主题/时间/项目组织知识库

    目标效果： - ✅ 用户可以选择导入到哪个 Graph - ✅ 支持创建新的 Graph（输入名称） - ✅ 清晰显示当前 Graph 和统计信息 - ✅ 不同项目/主题的数据完全隔离

    🏗️ 架构设计¶

    数据流¶

    前端 UI
  ↓
用户选择 Graph: "my_project"
  ↓
导入文档 (files/urls)
  ↓
API: POST /import { graph_name: "my_project", ... }
  ↓
后端创建/使用: my_project_vgrag_* (3个表)
  ↓
数据存储到指定的 Graph
  ↓
前端刷新 Graph 列表，显示新数据

    UI 流程¶

    1. 用户点击 "Import" 按钮
   ↓
2. 显示导入对话框
   ├─ Graph 选择器 ⭐ (新增)
   │   ├─ 下拉选择现有 Graph
   │   └─ "Create New" 选项
   ├─ 文件/URL 输入
   └─ 导入设置
   ↓
3. 用户选择/创建 Graph
   ↓
4. 上传文档
   ↓
5. 显示导入结果
   └─ 提示: "已导入到 {graph_name}"

  // 不能为空
  if (!name.trim()) return 'Name is required'

  // 只能包含字母、数字、下划线、连字符
  if (!/^[a-zA-Z0-9_-]+$/.test(name)) {
    return 'Only letters, numbers, underscores, and hyphens allowed'
  }

  // 不能与现有 Graph 重名
  if (existingGraphs.includes(name)) {
    return 'Graph already exists'
  }

  // 长度限制
  if (name.length > 50) return 'Name too long (max 50 characters)'

  return null
}
